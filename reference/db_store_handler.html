app_pkg.functions.db_store_handler

app_pkg.functions.db_store_handler.db_store_handler(event: Event, output_queue: Queue, root_dir: str) → int

Función para asociar al evento C-STORE en el DICOM Storage Service Class Provider. Para cada objeto DICOM recibido:

  • Extrae y decodifica el objeto DICOM enviado.

  • Verifica que el campo SOPClassUID sea 1.2.840.10008.5.1.4.1.1.128 (Positron Emission Tomography Image Storage); de lo contrario, lo ignora.

  • Verifica que estén presentes todos los campos mandatorios (ver extract_from_dataset).

  • Guarda el archivo en disco (si no existe ya) y actualiza la base de datos (ver store_dataset).

  • Pone en la queue.Queue de salida un dict con la siguiente estructura:

    • dataset: Dataset new_ds generado por extract_from_dataset.

    • recon_ds: Dataset recon_ds generado por extract_from_dataset.

    • address: str con la dirección IP del dispositivo que inició la asociación y envío los DICOMS.

    • ae_title: str con el AE Title de la aplicación que inició la asociación y envío los DICOMS.

Parámetros:

  • event – Event pasado al handler cuando se recibe un nuevo objeto DICOM para almacenar.

  • output_queue – queue.Queue donde se pasarán los resultados al próximo paso de procesamiento.

  • root_dir – raíz del directorio donde se guardarán los objetos DICOM recibidos.

Devuelve:

0xC210 si el objeto DICOM no tiene SOP Class UID o no se puede decodificar.

Devuelve:

0xA700 si el objeto DICOM no tiene algún campo mandatorio.

Devuelve:

0xA700 si ocurre un error al guardar el objeto DICOM en disco o en base de datos.

Devuelve:

0x0000 si el objeto DICOM se guardo correctamente o si fue ignorado por SOP Class UID incorrecto.

app_pkg.functions.db_store_handler.extract_from_dataset(ds: Dataset | str) → tuple[Dataset, Dataset]

Verifica que un objeto DICOM contenga ciertos campos mandatorios y los extrae junto con otros opcionales.

Campos mandatorios:

  • StudyInstanceUID

  • SeriesInstanceUID

  • SOPInstanceUID

  • ImagePositionPatient

Parámetros:

ds – Dataset para procesar, o str con path al archivo DICOM.

Devuelve:

tuple con dos Dataset (new_ds, recon_ds) con los siguientes campos (si están disponibles):

new_ds:

  • StudyInstanceUID

  • SeriesInstanceUID

  • SOPInstanceUID

  • ImagePositionPatient

  • NumberOfSlices

  • PatientName

  • StudyDate

  • SeriesDescription

recon_ds:

  • PixelSpacing

  • ReconstructionMethod

  • Manufacturer

  • ManufacturerModelName

  • SliceThickness

  • ConvolutionKernel

  • PatientWeight

  • ActualFrameDuration

  • RadiopharmaceuticalInformationSequence

  • 0x000910B3

  • 0x000910B2

  • 0x000910BA

  • 0x000910BB

  • 0x000910DC

Raises:

AttributeError si falta alguno de los campos mandatorios.

app_pkg.functions.db_store_handler.store_dataset(ds: Dataset, root_dir: str) → Literal[-1, 0, 1]

Genera o actualiza en base de datos los objetos correspondientes a un objeto DICOM:

  • Patient

  • Study

  • Series

  • Instance

Si la instancia no existía previamente en base de datos, además se guarda el objeto DICOM en el almacenamiento.

Parámetros:

  • ds – Dataset para procesar.

  • root_dir – raíz del directorio en el disco donde se guardarán los archivos DICOM.

Devuelve:

1 si la instancia ya existía y la actualización de la base de datos fue exitosa.

Devuelve:

0 si la instancia no existía y las operaciones en base de datos y en disco fueron exitosas.

Devuelve:

-1 si ocurre algún error en las operaciones en base de datos o en disco.

app_pkg.functions.db_store_handler.db_create_update_patient(ds: Dataset) → Patient

Genera o actualiza en base de datos un objeto Patient a partir de la información contenida en el Dataset de entrada. Si ya existe un objeto con ese PatientID, se actualiza el PatientName del objeto ya existente.

Parámetros:

  • ds – Dataset para procesar.

  • root_dir – raíz del directorio en el disco donde se guardarán los archivos DICOM.

Devuelve:

Patient con el objeto generado o modificado.

app_pkg.functions.db_store_handler.db_create_update_study(ds: Dataset, path: str = None) → Study

Genera o actualiza en base de datos un objeto Study a partir de la información contenida en el Dataset de entrada. Si ya existe un objeto con ese StudyInstanceUID, se actualizan los campos StudyDate, StudyDescription, PatientWeight, PatientSize, y PatientAge del objeto ya existente.

Parámetros:

  • ds – Dataset para procesar.

  • path – directorio donde se almacena el estudio en disco.

Devuelve:

Study con el objeto generado o modificado.

app_pkg.functions.db_store_handler.db_create_update_series(ds: Dataset, path: str = None) → Series

Genera o actualiza en base de datos un objeto Series a partir de la información contenida en el Dataset de entrada. Si ya existe un objeto con ese SeriesInstanceUID, se actualizan los campos SeriesDate, SeriesTime, SeriesDescription, Modality, y SeriesNumber del objeto ya existente.

Parámetros:

  • ds – Dataset para procesar.

  • path – directorio donde se almacena la serie en disco.

Devuelve:

Series con el objeto generado o modificado.

app_pkg.functions.db_store_handler.db_create_instance(ds: Dataset, filename: str) → Instance

Genera en base de datos un objeto Instance a partir de la información contenida en el Dataset de entrada. Genera ValueError si ya existe un objeto con ese SOPInstanceUID en base de datos.

Parámetros:

  • ds – Dataset para procesar.

  • filename – directorio donde se almacena la instancia en disco.

Devuelve:

Instance con el objeto generado.

Raises:

ValueError si la instancia ya existe.
