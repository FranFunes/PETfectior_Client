DICOM Storage Service Class Provider

Este servicio se implementa mediante la siguiente clase, que a su vez se configura usando un handler adecuado para el evento C-STORE:

class app_pkg.services.store_scp.StoreSCP(input_queue: Queue, c_store_handler: Callable, ae_title: str = 'PETFECTIOR', store_dest: str = 'incoming', *args, **kwargs)

Esta clase permite crear y manejar un servicio DICOM Store Service Class Provider, para poder recibir objetos DICOM de otros equipos. Hereda de pynetdicom.AE , por lo que comparte muchas de sus funcionalidades.

Aquí se describen solo aquellos atributos/métodos que se agregan a dicha clase.

__init__(input_queue: Queue, c_store_handler: Callable, ae_title: str = 'PETFECTIOR', store_dest: str = 'incoming', *args, **kwargs) → None
Parámetros:

  • input_queue – queue.Queue donde se pondrán los objetos Dataset generados para su posterior procesamiento.

  • c_store_handler – función de procesamiento de cada Dataset recibido.

  • store_dest – raíz de la estructura de directorios donde se almacenarán los Dataset.

echo(device: dict) → int | Dataset

Verifica la conexión con un dispositivo DICOM remoto, mediante un mensaje DICOM C-ECHO.

Devuelve:

-1 si la conexión no fue exitosa

Devuelve:

Dataset con la respuesta del dispositivo remoto, si la conexión fue exitosa

Para controlar cómo una Application Entity maneja los diferentes eventos DICOM que ocurren durante una asociación, pynetdicom permite vincular un handler a cada tipo de evento (ver Storage SCP ). En nuestro caso, se usa la función db_store_handler, cuya documentación se puede revisar para más detalles.
